# Требования к реализации

Новые словари реализовывать как наследники `IDictionary` с методами:

| Метод            | Описание                                                                                                                                         |
|------------------|--------------------------------------------------------------------------------------------------------------------------------------------------|
| GetValue         | Возвращает значение по указанному ключу. Если ключ отсутствует в словаре, генерируется `std::out_of_range`.                                      |
| InsertOrAssign   | Если ключ уже существует, его значение заменяется на новое. Если ключ отсутствует, создаётся новая пара ключ-значение.                           |
| ContainsKey      | Проверяет наличие указанного ключа в словаре. Возвращает `true`, если ключ найден, иначе `false`.                                                |
| TryGetValue      | Пытается получить значение по ключу. Если ключ найден - записывает значение в переданный параметр и возвращает `true`, иначе возвращает `false`. |
| Add              | Добавляет новую пару ключ-значение. Если ключ уже существует, генерируется `std::invalid_argument`.                                              |
| Remove           | Удаляет элемент по ключу. Возвращает `true`, если элемент был найден и удалён, иначе `false`.                                                    |
| Clear            | Удаляет все элементы словаря. После выполнения словарь становится пустым.                                                                        |
| Count            | Возвращает количество элементов в словаре.                                                                                                       |
| Items            | Возвращает константную ссылку на контейнер, содержащий все пары ключ-значение.                                                                   |
| initializer_list | Конструктор, позволяющий инициализировать словарь списком пар `{key, value}`. При повторяющихся ключах генерировать `std::invalid_argument`.     |

Вручную перегружать  `operator[]` не нужно, реализация берется из интерфейса и использует `GetValue` наследника. По итогу получается следующее поведение:
```c++
#include "Dictorium/Dictorium.h"

using namespace dtr;

int main() {
  LinearDictionary<std::string, int> dict = {
    {"тест1", 1},
    {"тест2", 2},
  };
  dict["тест3"] = 4; // исключения нет, значение сохраняется
  std::cout << dict["тест4"]; // std::out_of_range("Key not found")
}
```

Для `ToString()`, `operator<<` и `WriteToStream` автоматически  берется реализация из интерфейса, ожидая реализованный `_getItemsEnumerator()`. Что позволяет удобно просматривать содержимое всем наследникам `IDictionary`:
```c++
#include "Dictorium/Dictorium.h"

using namespace dtr;

int main() {
  LinearDictionary<int, std::string> dict = {
    {1, "test1"},
    {2, "test2"}
  };
  std::cout << dict;
}
```
Вывод:
```json
{
    "1": "test1",
    "2": "test2"
}
```
