# AvlDictionary

`AvlDictionary<TKey, TValue>` — динамический словарь на основе **AVL-дерева** (Adelson-Velsky & Landis, 1962). Гарантирует высоту не более `1.44 × log₂(n)` в любой момент времени: после каждой вставки и удаления дерево автоматически восстанавливает баланс через однократный или двукратный поворот.

Все операции вставки, удаления и поиска выполняются за **строго O(log n)** — без амортизации и без вырождения в линейный случай, характерного для наивного BST.

Ключ должен поддерживать полный порядок сравнения (`std::totally_ordered`): именно он определяет структуру дерева. Итерация выполняется в порядке возрастания ключей (in-order обход).

---

## Сложности операций

| Метод                          | O (верхняя граница) | Ω (нижняя граница) | Пояснение                                                                              |
|--------------------------------|---------------------|---------------------|----------------------------------------------------------------------------------------|
| `ContainsKey`                  | O(log n)            | Ω(1)                | Итеративный спуск по дереву; Ω(1) — ключ в корне                                      |
| `TryGetValue`                  | O(log n)            | Ω(1)                | Аналогично `ContainsKey`, дополнительно копирует значение                              |
| `GetValue`                     | O(log n)            | Ω(1)                | Бросает `std::out_of_range`, если ключ не найден                                       |
| `Add`                          | O(log n)            | Ω(log n)            | Спуск + обратный проход с пересчётом высот и балансировкой; бросает `invalid_argument` при дубликате |
| `InsertOrAssign` (новый ключ)  | O(log n)            | Ω(log n)            | Вставка с балансировкой                                                                |
| `InsertOrAssign` (ключ есть)   | O(log n)            | Ω(1)                | Спуск до узла, обновление значения на месте; поворотов нет                             |
| `Remove`                       | O(log n)            | Ω(log n)            | Спуск + удаление + обратный проход с балансировкой                                     |
| `Clear`                        | O(n)                | Ω(n)                | Рекурсивный обход и освобождение всех узлов                                            |
| `Count`                        | O(1)                | Ω(1)                | Возвращает `_count` напрямую                                                           |
| `Height`                       | O(1)                | Ω(1)                | Читается из поля `height` корневого узла                                               |
| `initializer_list` конструктор | O(n log n)          | Ω(n log n)          | n вызовов `Add`                                                                        |
| Итерация (`begin`/`end`)       | O(n)                | Ω(n)                | In-order обход через стек; каждый `operator++` амортизированно O(1)                   |

> **Обозначение:** n — текущее количество элементов в словаре.

## Устройство узла

Каждый элемент хранится в узле `detail::AvlNode<TKey, TValue>`:

```
┌──────────────────────────┐
│  data: pair<TKey,TValue> │  ← пара ключ-значение
│  left:  AvlNode*         │  ← левый потомок (ключи < data.first)
│  right: AvlNode*         │  ← правый потомок (ключи > data.first)
│  height: int             │  ← высота поддерева с корнем в этом узле
└──────────────────────────┘
```

`height` пересчитывается за O(1) после каждой операции по формуле:
```
height(node) = 1 + max(height(left), height(right))
```
Пустой узел имеет высоту 0.

---

## Балансировка

**Баланс-фактор** узла — разность высот левого и правого поддеревьев:
```
bf(node) = height(left) - height(right)
```

AVL-инвариант: `|bf(node)| ≤ 1` для каждого узла. При нарушении (bf = ±2) выполняется один из четырёх случаев:

### Left-Left (bf = +2, bf(left) ≥ 0) — один правый поворот

```
      z (+2)            y
     / \              /   \
    y   C    →       x     z
   / \              / \   / \
  x   B            A   B B   C
```

### Left-Right (bf = +2, bf(left) < 0) — левый поворот left, затем правый поворот z

```
    z (+2)         z (+2)          x
   / \            / \            /   \
  y   C   →      x   C   →     y     z
 / \            / \           / \   / \
A   x          y   C'        A   B B'  C
   / \        / \
  B   C'     A   B
```

### Right-Right (bf = −2, bf(right) ≤ 0) — один левый поворот (симметрично LL)

### Right-Left (bf = −2, bf(right) > 0) — правый поворот right, затем левый поворот z (симметрично LR)

После каждого поворота высоты затронутых узлов пересчитываются снизу вверх: сначала опустившийся узел, затем новый корень поддерева.

## Вставка (`Add` / `InsertOrAssign`)

```
1. Рекурсивный спуск: key < node.key → влево, key > node.key → вправо.
2. Если ключ совпал:
   - Add:            бросить std::invalid_argument.
   - InsertOrAssign: обновить значение, вернуть node (поворотов нет).
3. Если node == nullptr: создать новый AvlNode, вернуть его.
4. На обратном пути вверх:
   - _updateHeight(node)
   - node = _balance(node)   ← 0 или 1 поворот
5. Вернуть новый корень поддерева.
```

---

## Удаление (`Remove`)

```
1. Рекурсивный спуск до целевого узла.
2. Целевой узел найден:
   a. Нет потомков / один потомок → вернуть единственного потомка (или nullptr).
   b. Два потомка:
      - Найти in-order преемника (минимум правого поддерева).
      - Скопировать data преемника в текущий узел.
      - Рекурсивно удалить преемника из правого поддерева.
3. На обратном пути вверх:
   - _updateHeight(node)
   - node = _balance(node)
```

Случай (b) гарантирует, что при замене данными преемника AVL-инвариант восстанавливается корректно: преемник всегда является листом или узлом с одним правым потомком.

## Итератор

`AvlDictionary::Iterator` — однонаправленный forward-итератор, реализующий in-order обход без рекурсии через явный стек.

**Инициализация (конструктор от корня):**
```
_pushLeft(root)   ← все узлы по левой ветке от root
_advance()        ← взять верхушку стека как _current
```

**`operator++`:**
```
_pushLeft(_current->right)   ← зайти в правое поддерево
взять следующий элемент из стека как _current
```

Инициализация — O(h) = O(log n). Каждый `operator++` амортизированно O(1): каждый узел заталкивается и выталкивается из стека ровно один раз.

Разыменование возвращает `std::pair<TKey, TValue>&`, поэтому совместимо с range-for и с `_writeItems` из `IDictionary`.

## Примеры использования

```cpp
#include "Dictorium/Dictorium.h"

using namespace dtr;

int main() {
    AvlDictionary<std::string, int> dict = {
        {"cherry", 3},
        {"apple",  1},
        {"banana", 2}
    };

    // Поиск
    std::cout << dict["apple"] << '\n';          // 1

    // Обновление
    dict.InsertOrAssign("apple", 100);
    std::cout << dict.GetValue("apple") << '\n'; // 100

    // Удаление
    dict.Remove("banana");
    std::cout << dict.ContainsKey("banana") << '\n'; // false

    // Высота
    std::cout << dict.Height() << '\n';          // 2

    // In-order итерация (apple → cherry)
    for (auto& [k, v] : dict)
        std::cout << k << ": " << v << '\n';

    // Вывод словаря
    std::cout << dict;
}
```

## Замеры производительности

Сравнение с `std::map` (красно-чёрное дерево), N = 500 000 ключей типа `int`, Release, MinGW GCC 15.2, Windows.

| Операция            | `std::map` | `AvlDictionary` | Быстрее          |
|---------------------|-----------:|----------------:|------------------|
| Инициализация       |   ~388 ms  |        ~385 ms  | ≈ одинаково      |
| Чтение (GetValue)   |   ~700 нс  |        ~470 нс  | AvlDictionary ×1.5 |
| ContainsKey         |   ~710 нс  |        ~510 нс  | AvlDictionary ×1.4 |
| Итерация (полная)   |   ~58 ms   |        ~17 ms   | AvlDictionary ×3.4 |
| Высота дерева       |   ~30–32   |            23   | AVL строже       |

**Инициализация одинакова** — оба дерева выполняют n вставок с O(log n) каждая; разница в стратегии балансировки нивелируется на Windows heap.

**Чтение быстрее** — AVL гарантирует высоту ≤ `1.44 × log₂(n)` против `2 × log₂(n)` у красно-чёрного дерева; при N = 500 000 это 23 против ~31 уровня → меньше сравнений и промахов кэша на пути поиска.

**Итерация быстрее в ~3.4x** — при последовательном in-order обходе узлы `AvlDictionary` оказываются ближе друг к другу в памяти (аллокационная локальность), чем хаотично разбросанные узлы `std::map`.

## Ограничения

- Ключ должен удовлетворять `std::totally_ordered` — дерево поиска требует строгого порядка.
- Память: O(n), каждый узел выделяется отдельно через `new`; нет пула аллокации.
- Итератор — только forward; обратный обход не поддерживается.
- Нет `const`-итератора: `begin()`/`end()` возвращают `Iterator` по константной ссылке на дерево, но разыменование даёт неконстантную пару. Для защиты значений используйте `GetValue(key) const`.

---

## Итог

`AvlDictionary` обеспечивает **гарантированный O(log n)** для всех операций изменения и поиска без амортизации. По сравнению со стандартным `std::map` выигрывает на чтении (~1.4–1.5x) и итерации (~3x) за счёт более строгого баланса и лучшей локальности памяти. Рекомендуется в сценариях, где операции чтения и итерации доминируют над вставкой.
