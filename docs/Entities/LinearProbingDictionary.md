# LinearProbingDictionary

`LinearProbingDictionary<TKey, TValue, Hash>` — реализация словаря поверх `std::vector<Entry>` с открытой адресацией и линейным пробированием. Удаление реализовано через tombstone-маркеры (`EntryState::Deleted`).

| Метод            | Сложность | Аморт. сложность | Пояснение |
|------------------|-----------|------------------|-----------|
| GetValue         | O(1)      | O(1)             | `_findIndex` → линейный проход от стартового индекса; O(1) в среднем при низком коэффициенте загрузки; O(n) в худшем случае |
| InsertOrAssign   | O(1)      | O(1)             | `_ensureCapacity` + `_findSlotForInsert` → O(1) в среднем; рехэширование добавляет O(1) амортизированно |
| ContainsKey      | O(1)      | O(1)             | Делегирует в `_findIndex` → O(1) в среднем |
| TryGetValue      | O(1)      | O(1)             | `_findIndex` → O(1) в среднем; при нахождении копирует значение |
| Add              | O(1)      | O(1)             | `_ensureCapacity` → O(1) аморт. + `_findSlotForInsert` → O(1) в среднем; бросает `std::invalid_argument` при дубликате |
| Remove           | O(1)      | O(1)             | `_findIndex` → O(1) в среднем; запись tombstone — O(1); физического сдвига нет |
| Clear            | O(m)      | O(m)             | `vector::clear` + `resize(8)` — уничтожение всех `m` ячеек таблицы |
| Count            | O(1)      | O(1)             | Возвращает поле `_count` |
| initializer_list | O(n)      | O(n)             | `n` вызовов `Add` → O(1) каждый; суммарное рехэширование не превышает O(n) |

> **Худший случай** для операций поиска/вставки/удаления — **O(n)**. Возникает при высокой фрагментации таблицы (накопление tombstone-ячеек) или вырождении хэш-функции. При коэффициенте загрузки ≤ 0.75 и равномерном распределении хэшей среднее число пробирований стремится к константе.

---

# Назначение

`LinearProbingDictionary` хранит все данные в **едином плоском массиве** без динамических выделений памяти на элемент, что обеспечивает лучшую кэш-локальность по сравнению с методом цепочек.

Tombstone-удаление (`Deleted`) позволяет не перестраивать цепочки пробирования, но накапливает мусор в таблице. Рехэширование автоматически очищает tombstone-ячейки — они не переносятся в новую таблицу.

Итератор обходит весь массив за **O(n + m)**, пропуская `Empty` и `Deleted` ячейки.
