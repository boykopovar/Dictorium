# PerfectHashDictionary

`PerfectHashDictionary<TKey, TValue>` - статический словарь с **совершенным хешированием** (Perfect Hashing). Построение выполняется один раз при инициализации; все последующие операции чтения работают за **строго O(1)** / Ω(1) (никаких амортизированных оценок).

Реализует двухуровневую схему хеширования:
- **Первый уровень** - глобальный хеш распределяет ключи по корзинам.
- **Второй уровень** - каждая корзина получает собственный seed, гарантирующий отсутствие коллизий внутри неё.

Все элементы хранятся в едином плоском массиве `_values`; обращение к любому ключу требует ровно двух хеш-вычислений и одного обращения к памяти.

## Сложности операций

| Метод                              | O (верхняя граница) | Ω (нижняя граница) | Пояснение                                                                                                                       |
|------------------------------------|---------------------|--------------------|---------------------------------------------------------------------------------------------------------------------------------|
| `ContainsKey`                      | O(1)                | Ω(1)               | Два хеш-вычисления + проверка `Exists` и совпадения ключа                                                                       |
| `TryGetValue`                      | O(1)                | Ω(1)               | Аналогично `ContainsKey`, дополнительно копирует значение. Не выполняет сравнение ключа (ожидается ключ из исходного множества) |
| `TryGetValidatedValue`             | O(1)                | Ω(1)               | Аналогично `ContainsKey`, дополнительно копирует значение и выполняет сравнение ключа                                           |
| `GetValue`                         | O(1)                | Ω(1)               | Не проверяет совпадение ключа; ожидается ключ из исходного множества                                                            |
| `GetValidatedValue`                | O(1)                | Ω(1)               | Полная проверка ключа; бросает исключение при несовпадении                                                                      |
| `Count`                            | O(1)                | Ω(1)               | Возвращает `_count` напрямую                                                                                                    |
| `Remove`                           | O(1)                | Ω(1)               | Логическое удаление (tombstone): сбрасывает флаг `Exists`, физически запись остаётся                                            |
| `Clear`                            | O(n)                | Ω(n)               | Очищает `_buckets` и `_values`; обходит весь плоский массив                                                                     |
| `InsertOrAssign` (ключ существует) | O(1)                | Ω(1)               | Ключ найден за O(1); значение обновляется на месте                                                                              |
| `InsertOrAssign` (новый ключ)      | O(n^1.5)            | Ω(n)               | Требует полной перестройки                                                                                                      |
| `Add` (ранее удалённый ключ)       | O(1)                | Ω(1)               | Слот совпадает по первому ключу → восстановление без перестройки                                                                |
| `Add` (новый ключ)                 | O(n^1.5)            | Ω(n)               | Требует полной перестройки; собирает все живые пары и вызывает `_build`                                                         |
| `initializer_list` / конструктор   | O(n^1.5)            | Ω(n)               | Работает с initializer_list и указателями на диапазон. Подробнее - в разделе «Построение».                                      |
| Итерация (`begin`/`end`)           | O(n)                | Ω(n)               | Весь проход обычно O(2n) за счет пропуска слотов `Exists = false`                                                               |

> **Обозначения:** n - количество хранимых ключей; m - размер плоского массива `_values` (m ≥ n, в среднем m ≈ 2n при успешном построении).

## Построение (`_build`)

Построение - самая затратная операция. Понимание её механики важно для корректного применения словаря.

### Алгоритм (пошагово)

```
1. Выбрать случайный globalSeed.
2. Распределить все n пар по _tableSize корзин через _hash(key, globalSeed, _tableSize).
3. Проверить: Σ |bucket_i|² ≤ 2·_tableSize (условие «малых столкновений»).
   Если нет - повторить с шага 1.
4. Для каждой непустой корзины с |bucket| > 1:
   a. Задать размер вторичной таблицы: p = nextPrime(|bucket|²).
   b. Подобрать локальный seed (до 1000 попыток), при котором
      все ключи корзины размещаются без коллизий.
   c. Если seed не найден - удвоить _tableSize, повторить с шага 1.
5. Вычислить Offset для каждой корзины (prefix sum по Size).
6. Заполнить плоский массив _values.
```

| Фаза                      | O             | Ω        | Пояснение                                                                                                                                                                         |
|---------------------------|---------------|----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Глобальная рассылка       | O(n)          | Ω(n)     | Один проход по всем ключам                                                                                                                                                        |
| Проверка Σ\|bucket\|²     | O(n)          | Ω(n)     | Один проход по корзинам; _tableSize = n на старте                                                                                                                                 |
| `_nextPrime`              | O(n · ln²(n)) | Ω(1)     | Поиск ближайшего простого на числа. На практике обычно требует несколько десятков операций, но формально является самым узким местом алгоритма, не давая свести построение к O(n) |
| Поиск localSeed           | O(n)          | Ω(1)     | В худшем случае одна корзина содержит 2n ключей - O(n)                                                                                                                            |
| Число глобальных итераций | O(√n)         | Ω(1)     | Теоретический максимум до выполнения условия Σ\|bucket\|² ≤ 2n; Ω(1) — с первой попытки                                                                                           |
| **Итого**                 | **O(n^1.5)**  | **Ω(n)** |                                                                                                                                                                                   |

> На практике условие Σ|bucket|²  ≤ 2n отсеивает вырожденные распределения ещё до поиска localSeed, поэтому реальное поведение близко к O(n) на итерацию. Строгая верхняя граница O(n^2.5) достигается лишь при патологических входных данных и почти никогда не реализуется на случайных ключах.
Константа при O(n^1.5) велика на практике из-за случайного поиска seed. `_findSeed` обычно завершается за 1 итерацию, в тестах был зафиксирован рекорд 9 итераций. Глобальный цикл обычно завершается за 1 итерацию, в тестах был зафиксирован рекорд 4 итерации.

### Граничные случаи

- Пустой набор ключей → `std::invalid_argument`.
- Дублирующиеся ключи → `std::invalid_argument` (обнаруживается на шаге 4).
- Превышение `PERFHASH_MAX_ATTEMPTS` глобальных итераций → `std::runtime_error`.

## Особенности и ограничения

### Статичность ключевого множества

`PerfectHashDictionary` оптимизирован для **неизменяемого** множества ключей. Добавление нового ключа (не существовавшего при построении) требует полной перестройки O(n^1.5). Соответствующие методы помечены `[[deprecated]]`.

### Логическое удаление (`Remove`)

`Remove` не удаляет элемент физически - он выставляет `Exists = false` (tombstone). Слот по-прежнему занят. Если впоследствии вызвать `Add` с тем же ключом, перестройки **не будет**: ключ попадёт в тот же слот, `Exists` будет возвращён в `true`.

### `GetValue` vs `GetValidatedValue`

|                            | `GetValue` / `TryGetValue`                             | `GetValidatedValue` / `TryGetValidatedValue` |
|----------------------------|--------------------------------------------------------|----------------------------------------------|
| Проверяет совпадение ключа | ❌                                                      | ✅                                            |
| Поведение при чужом ключе  | UB / неверное значение / ложно-положительный результат | `std::out_of_range`                          |
| Когда использовать         | Ключ гарантированно из исходного множества             | Неизвестный или недоверенный ключ            |

### Потребление памяти

Плоский массив `_values` имеет размер m ≈ 2n ожидаемо (из-за вторичных таблиц размером `nextPrime(|bucket|²)`). Дополнительно хранится `_buckets` размером `_tableSize` ≈ n. Итого: **≈ 3n** записей (`PhSlot` + `PhBucket`).

## Примеры использования

```c++
#include "Dictorium/Dictorium.h"

using namespace dtr;

int main() {
  PerfectHashDictionary<std::string, int> dict = {
    {"apple", 1},
    {"banana", 2},
    {"cherry", 3}
  };

  std::cout << dict["banana"] << '\n';

  // O(1) проверка наличия
  std::cout << dict.ContainsKey("apple") << '\n'; // true

  // O(1) логическое удаление
  dict.Remove("cherry");
  std::cout << dict.ContainsKey("cherry") << '\n'; // false

  // Восстановление удалённого ключа - O(1), без перестройки
  dict.Add("cherry", 99);
  
  // Добавление НОВОГО ключа - O(n^1.5), перестройка
  dict.Add("date", 4); // вызывает _build
  
  std::cout << dict;

}
```

## Итог

`PerfectHashDictionary` обеспечивает **гарантированное O(1) / Ω(1)** для всех операций чтения над фиксированным множеством ключей. Платой является высокая стоимость построения O(n^1.5) и отсутствие поддержки динамического изменения ключевого множества без полной перестройки.

Словарь рекомендуется применять в сценариях, где набор ключей известен заранее и остаётся неизменным на протяжении времени жизни объекта.
