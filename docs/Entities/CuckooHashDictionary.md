# CuckooHashDictionary

`CuckooHashDictionary<TKey, TValue>` — динамический словарь на основе **кукушкиного хеширования** (Cuckoo Hashing). Использует две независимые хеш-таблицы с разными seed-ами; каждый ключ хранится ровно в одном из двух возможных слотов, что даёт **гарантированное O(1)** для всех операций чтения.

---

## Сложности операций

| Метод                                          | O           | Ω    | Пояснение                                                                               |
|------------------------------------------------|-------------|------|-----------------------------------------------------------------------------------------|
| `ContainsKey`                                  | O(1)        | Ω(1) | Два хеш-вычисления, два обращения к памяти                                              |
| `TryGetValue`                                  | O(1)        | Ω(1) | Аналогично `ContainsKey`, дополнительно копирует значение                               |
| `GetValue`                                     | O(1)        | Ω(1) | Бросает `out_of_range` если ключ не найден                                              |
| `Count`                                        | O(1)        | Ω(1) | Возвращает `_keysCount` напрямую                                                        |
| `Remove`                                       | O(1)        | Ω(1) | Физическое удаление: сбрасывает `Exists`, уменьшает счётчик                             |
| `Clear`                                        | O(n)        | Ω(n) | Очищает обе таблицы, сбрасывает seed-ы                                                  |
| `Add`                                          | O(1) amorт. | Ω(1) | При превышении load factor или цикле кукушки — вызывает `Rehash`                        |
| `InsertOrAssign`                               | O(1) amorт. | Ω(1) | Аналогично `Add`, при совпадении ключа обновляет значение на месте                      |
| `Rehash`                                       | O(n)        | Ω(n) | Удваивает таблицы, переиндексирует все элементы с новыми seed-ами                       |
| Итерация (`begin`/`end`)                       | O(n)        | Ω(n) | Проход по двум таблицам с пропуском пустых слотов; реально O(2m) где m — размер таблицы |
| Конструктор из `initializer_list` / итераторов | O(n) amorт. | Ω(n) | Вызывает `Add` для каждого элемента                                                     |

> **Амортизированное O(1)** для `Add`/`InsertOrAssign`: рехеш происходит редко (при load factor > 0.5 или цикле кик), каждый рехеш удваивает ёмкость.

---

## Устройство структуры

Два независимых вектора `_table1` и `_table2` одинакового размера. Каждый ключ имеет ровно один кандидат-слот в каждой таблице:

```
index1 = hash1(key) → позиция в _table1
index2 = hash2(key) → позиция в _table2
```

При вставке, если оба слота заняты, элемент «выталкивает» жильца — отсюда название. Вытолкнутый элемент ищет место в своей второй таблице, и так далее. Если цепочка кик превышает `2·log2(tableSize)` — вызывается `Rehash`.

### Load factor и рехеш

```
load factor = keysCount / (table1.size() + table2.size())
порог       = 0.5
```

При превышении порога или неудаче вставки: seed-ы пересчитываются через mixing-функцию, таблицы удваиваются, все элементы переиндексируются.

---

## Remove vs PerfectHashDictionary

В отличие от `PerfectHashDictionary`, удаление **физическое** — слот освобождается и может быть занят другим ключом. Нет tombstone-проблемы, нет накопления мёртвых слотов.

## Параметры по умолчанию

| Константа                    | Значение | Назначение                                            |
|------------------------------|----------|-------------------------------------------------------|
| `DTR_CUCKOO_MAX_LOAD_FACTOR` | `0.5`    | Порог заполненности перед рехешем                     |
| `DTR_CUCKOO_INIT_CAPACITY`   | `8`      | Начальный размер каждой таблицы                       |
| `DTR_CUCKOO_KICKS_FACTOR`    | `2.0`    | Множитель для максимального числа кик: `2·log2(size)` |

## Пример использования

```cpp
#include "Dictorium/Dictorium.h"
using namespace dtr;

int main() {
    CuckooHashDictionary<std::string, int> dict = {
        {"apple", 1},
        {"banana", 2},
        {"cherry", 3}
    };

    std::cout << dict["banana"] << '\n';         // 2

    dict.Add("date", 4);
    dict.InsertOrAssign("apple", 99);

    std::cout << dict.ContainsKey("cherry") << '\n'; // true
    dict.Remove("cherry");
    std::cout << dict.ContainsKey("cherry") << '\n'; // false

    for (auto& [key, value] : dict) {
        std::cout << key << ": " << value << '\n';
    }
}
```

---

## Когда использовать

`CuckooHashDictionary` подходит когда набор ключей **динамически меняется** и важно гарантированное O(1) для чтения. Если ключи известны заранее и не меняются — предпочтительнее `PerfectHashDictionary`, дающий меньшую константу при чтении за счёт отсутствия двойной проверки.
