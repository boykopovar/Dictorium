# Алгоритм `Perfect Hashing`

## Построение
- Входные данные (пары `ключ-значение`):
```json
{
  "key1": 1,
  "key2": 2,
  "key3": 3,
  "key4": 4
}
```

`count` - Количество ключей

- Функция `int _hash(key, int seed, int tableSize)` для различных наборов `key, seed` обычно дает разный результат (не всегда).
- Функция `int _findSeed(keys, int size)` находит для набора ключей такой `seed`, при котором для каждого ключа из `keys` функция `_hash` дает уникальный результат.


### Шаги алгоритма:
1. Создается массив `buckets`, размером `count`. Где каждый элемент - корзина (массив) с максимальным размером count.
2. Выбирается случайный `globalSeed` (целочисленный).
3. Функция `_hash()` применяется к каждому ключу с параметрами: `seed=globalSeed`, `tableSize=count`
4. Каждый ключ помещается в корзину, индексом которой в массиве `buckets` является результат функции `_hash()`.
5. Если сумма квадратов получившихся размеров всех корзин превышает `2*count`, очистить корзины и продолжить с шага 2.
6. Для каждой корзины вызвать `_findSeed()` с параметром `size=bucketSize`, где `bucketSize` - фактическое количество элементов в корзине, и сохранить полученное значение `localSeed`.
7. В каждой корзине расположить каждый элемент на позицию `bucket[_hash(ley, localSeed, bucketSize)]`.


### Сложность построения

`_findSeed` для корзины размером `b` проверяет каждый элемент за `b` итераций:

```
T_findSeed(b) = O(b)
```

Шаг 5 отвергает `globalSeed` если `∑ bₖ² > 2n`, что гарантирует `∑ bₖ ≤ n`. Суммируя по всем корзинам:

```
T_build(n) = ∑ T_findSeed(bₖ) = ∑ O(bₖ) = O(∑ bₖ) = O(n)
```

| Нотация         | Запись | Смысл                               |
|-----------------|--------|-------------------------------------|
| Верхняя граница | O(n)   | ∃ c > 0, ∀n: T(n) ≤ c * n           |
| Нижняя граница  | Ω(n)   | ∃ c > 0, ∀n: T(n) ≥ c * n           |
| Точная оценка   | Θ(n)   | ∃ c₁,c₂ > 0, ∀n: c₁*n ≤ T(n) ≤ c₂*n |

> Построение - Θ(n) в среднем.


> Описанный алгоритм построения имеет сложность O(n) в общем случае. Но в реализации добавлены оптимизации, которые на практике увеличивают скорость, но верхняя оценка O будет выше.

## Доступ к элементу
- Входные данные: `key`

### Шаги алгоритма:
1. Получить значение `globalIndex` как результат функции `_hash(key, globalSeed, count)`.
2. Получить доступ к корзине `buckets[globalIndex]`.
3. Получить значение `localIndex` как результат функции `_hash(key, localSeed, bucketSize)`, где `localSeed` сохранен в корзине, а `bucketSize` - фактическое количество элементов в корзине.
4. Получить доступ к элементу корзины как `bucket[localIndex]` и вернуть значение.

### Сложность поиска

Поиск выполняет ровно два обращения к массиву - независимо от числа элементов `n`:

```
T(n) = a + b = const
```

Из этого следует:

| Нотация         | Запись   | Смысл                           |
|-----------------|----------|---------------------------------|
| Верхняя граница | O(1)     | ∃ c > 0, ∀n: T(n) ≤ c * 1       |
| Нижняя граница  | Ω(1)     | ∃ c > 0, ∀n: T(n) ≥ c * 1       |
| Точная оценка   | Θ(1)     | ∃ c₁,c₂ > 0, ∀n: c₁ ≤ T(n) ≤ c₂ |
| Строго быстрее  | o(log n) | lim(n→∞) T(n) / log n = 0       |

Θ(1) - одновременно является и O(1), и Ω(1).

> Поиск элемента - строго O(1) без ограничений общности. Так как это всегда только два доступа к массиву, без итераций.
