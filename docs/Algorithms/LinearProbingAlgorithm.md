# Алгоритм `LinearProbingDictionary`

`LinearProbingDictionary<TKey, TValue, Hash>` — реализация словаря на основе **хэш-таблицы с открытой адресацией и линейным пробированием**.

---

## Структура данных

- Входные данные (пары `ключ-значение`):
```json
{
    "key1": 1,
    "key2": 2,
    "key3": 3,
    "key4": 4
}
```

Таблица хранит плоский массив `_table` — `std::vector<Entry>`, где каждый элемент — структура:

```cpp
struct Entry {
    TKey key;
    TValue value;
    EntryState state; // Empty | Occupied | Deleted
};
```

- `_count` — количество активных пар (`Occupied`).
- `_maxLoadFactor = 0.75` — максимальный коэффициент загрузки. При превышении выполняется рехэширование.
- `Hash` — функтор хэширования (по умолчанию `std::hash<TKey>`).
- Стартовый индекс для любого ключа: `index = hasher(key) % _table.size()`.

### Состояния ячейки (`EntryState`)

| Состояние  | Смысл |
|------------|-------|
| `Empty`    | Ячейка никогда не использовалась. Поиск останавливается здесь. |
| `Occupied` | Ячейка содержит активную пару ключ-значение. |
| `Deleted`  | Ячейка была удалена (tombstone). Поиск продолжается через неё. |

---

## Построение (конструктор)

### Шаги алгоритма:
1. Создаётся вектор `_table` заданной ёмкости (по умолчанию `8`), все ячейки — `Empty`. `_count = 0`.
2. Для каждой пары `(key, value)` из входных данных вызывается метод `Add`.

> Построение из `n` элементов имеет сложность **O(n)** в среднем случае.

---

## Линейное пробирование

При коллизии (занятая ячейка) следующий кандидат выбирается по формуле:

```
pos = (startIndex + i) % capacity,  i = 0, 1, 2, ...
```

Поиск продолжается последовательно, пока не встретится `Empty` либо не будет обойдена вся таблица.

---

## Поиск слота (`_findIndex`)

Используется для **чтения** — `ContainsKey`, `TryGetValue`, `GetValue`, `Remove`.

### Шаги алгоритма:
1. Вычисляется стартовый индекс: `index = hasher(key) % capacity`.
2. Линейный проход по ячейкам:
   - `Empty` → ключ отсутствует, вернуть `capacity` (sentinel).
   - `Occupied` и `key == искомый` → вернуть `pos`.
   - `Deleted` → пропустить, продолжить проход.
3. Если обошли всю таблицу без результата — вернуть `capacity`.

> Tombstone-ячейки (`Deleted`) не прерывают поиск, чтобы цепочка пробирования оставалась непрерывной.

---

## Поиск слота для вставки (`_findSlotForInsert`)

Используется в `Add` и `InsertOrAssign`.

### Шаги алгоритма:
1. Вычисляется стартовый индекс.
2. Линейный проход по ячейкам:
   - `Occupied` и `key == искомый` → вернуть `pos` (ключ уже есть, обновление).
   - `Deleted` → запомнить первый такой слот в `firstDeleted` (продолжить проход — ключ может быть дальше).
   - `Empty` → если `firstDeleted` был сохранён — вернуть его (переиспользование tombstone); иначе вернуть `pos`.
3. Если обошли всю таблицу — вернуть `capacity`.

> Переиспользование `Deleted`-слотов под новую вставку сокращает фрагментацию таблицы.

---

## Удаление (`Remove`)

Удаление реализовано через **tombstone-маркер** — ячейка не очищается, а помечается `Deleted`.

### Шаги алгоритма:
1. `_findIndex(key)` — найти позицию ключа.
2. Если не найден — вернуть `false`.
3. Установить `_table[pos].state = Deleted`, `_count--`, вернуть `true`.

> Физически ячейка не освобождается. Это сохраняет корректность цепочек пробирования для других ключей, которые «проходили» через эту позицию при вставке.

---

## Рехэширование (`_rehash`)

Выполняется автоматически при превышении `_maxLoadFactor`.

### Шаги алгоритма:
1. Старый `_table` сохраняется в `old`, `_table` заменяется новым вектором размером `old.size() * 2`.
2. `_count` сбрасывается в `0`.
3. Для каждой `Occupied`-ячейки из `old` вызывается `InsertOrAssign` — tombstone-ячейки (`Deleted`) **не переносятся**, что автоматически очищает таблицу от мусора.

> Рехэширование имеет сложность **O(n)**. Амортизированно к каждой вставке добавляется **O(1)**.

---

## Итератор

Итератор (`Iterator`) обходит плоский массив `_table`, пропуская ячейки не в состоянии `Occupied`.

### Устройство:
- Хранит индекс `_index` и указатель на `_table`.
- Вспомогательный метод `_skip()` продвигает `_index` вперёд до ближайшей `Occupied`-ячейки.
- `operator++` увеличивает `_index` на 1, затем вызывает `_skip()`.

> Полный проход по всем элементам — **O(n + m)**, где `n` — количество элементов, `m` — размер таблицы.
