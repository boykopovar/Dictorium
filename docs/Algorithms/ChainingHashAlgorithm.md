# Алгоритм `ChainingHashDictionary`

`ChainingHashDictionary<TKey, TValue, Hash>` — реализация словаря на основе **хэш-таблицы с открытым хэшированием (метод цепочек)**.

---

## Структура данных

- Входные данные (пары `ключ-значение`):
```json
{
    "key1": 1,
    "key2": 2,
    "key3": 3,
    "key4": 4
}
```

Таблица хранит массив `_buckets` — вектор корзин (`std::vector<Bucket>`), где каждая корзина (`Bucket`) — это `std::list<std::pair<TKey, TValue>>`.

- `_count` — текущее количество пар ключ-значение в таблице.
- `_maxLoadFactor = 0.75` — максимально допустимый коэффициент загрузки. При его превышении выполняется рехэширование.
- `Hash` — функтор хэширования (по умолчанию `std::hash<TKey>`).
- Функция `size_t _getIndex(key)` возвращает индекс корзины: `hasher(key) % _buckets.size()`.

---

## Построение (конструктор)

### Шаги алгоритма:
1. Создаётся вектор `_buckets` заданной ёмкости (по умолчанию `8` корзин), `_count = 0`.
2. Для каждой пары `(key, value)` из входных данных вызывается метод `Add`.
3. `Add` вычисляет индекс корзины через `_getIndex(key)`.
4. Корзина по этому индексу проверяется на наличие ключа. Если ключ уже есть — бросается `std::invalid_argument`.
5. Новая пара добавляется в конец корзины, `_count++`.
6. Вызывается `_ensureCapacity()` — если `_count / _buckets.size() > 0.75`, выполняется рехэширование.

> Построение из `n` элементов имеет сложность **O(n)** в среднем случае.

---

## Рехэширование (`_rehash`)

Выполняется автоматически при превышении `_maxLoadFactor`.

### Шаги алгоритма:
1. Создаётся новый вектор `newBuckets` размером `_buckets.size() * 2`.
2. Для каждой пары `(key, value)` из всех корзин вычисляется новый индекс: `hasher(key) % newSize`.
3. Пара помещается в соответствующую корзину нового вектора.
4. `_buckets` заменяется на `newBuckets`.

> Рехэширование имеет сложность **O(n)**. Амортизированно к каждой вставке добавляется **O(1)**.

---

## Доступ к элементу (`GetValue`, `TryGetValue`, `ContainsKey`)

- Входные данные: `key`

### Шаги алгоритма:
1. Вычисляется индекс корзины: `index = hasher(key) % _buckets.size()`.
2. Выполняется линейный перебор элементов корзины `_buckets[index]`.
3. Если ключ найден — возвращается значение (или `true`). Иначе — исключение или `false`.

> В среднем случае корзина содержит **O(1)** элементов, поэтому операции выполняются за **O(1)**. В худшем случае (все ключи попадают в одну корзину) — **O(n)**.

---

## Удаление (`Remove`)

- Входные данные: `key`

### Шаги алгоритма:
1. Вычисляется индекс корзины через `_getIndex(key)`.
2. Корзина перебирается итератором. При нахождении ключа — элемент удаляется через `list::erase`, `_count--`, возвращается `true`.
3. Если ключ не найден — возвращается `false`.

> Сложность: **O(1)** в среднем случае (удаление из `std::list` по итератору — O(1)).

---

## Итератор

Итератор (`Iterator`) обходит **все непустые корзины** последовательно.

### Устройство:
- Хранит два итератора: `_outer` (по вектору корзин) и `_inner` (по элементам текущей корзины).
- Вспомогательный метод `_skipEmpty()` перемещает `_outer` вперёд до первой непустой корзины.
- `operator++` сначала двигает `_inner`, при достижении конца корзины — переходит к следующей непустой.

> Полный проход по всем элементам — **O(n + m)**, где `n` — количество элементов, `m` — количество корзин.

---

## Особенности реализации

- При `Add` рехэширование происходит **после** вставки, поэтому коэффициент загрузки может кратковременно превысить `_maxLoadFactor` ровно на одну вставку.
- `Clear()` сбрасывает таблицу к размеру `8` корзин.
- Поддерживается конструктор из диапазона итераторов `[first, last)`.
- Дублирующийся ключ в `initializer_list` или диапазоне — бросается `std::invalid_argument("Duplicate key")`.
